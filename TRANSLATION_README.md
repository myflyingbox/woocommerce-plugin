# Translation Management

## After updating PHP source code

Run the extraction pipeline to regenerate the POT, merge new strings into existing PO files, and recompile the MO:

```bash
./bin/translations extract
```

This runs `xgettext` on all PHP files (excluding `vendor/`, `node_modules/`, `includes/lib/php-lce/`, `bin/`), merges the result into `lang/my-flying-box-fr_FR.po`, and recompiles `lang/my-flying-box-fr_FR.mo`.

## Translating new strings

Open `lang/my-flying-box-fr_FR.po` in a PO editor (e.g. [Poedit](https://poedit.net/)) or a text editor. New strings have an empty `msgstr`:

```po
msgid "My new string"
msgstr ""
```

Fill in the `msgstr` values, then recompile the MO:

```bash
./bin/translations mo
```

## Checking translation status

```bash
./bin/translations report
```

## Files

| File | Purpose |
|------|---------|
| `lang/my-flying-box.pot` | Source template — regenerated by `extract` |
| `lang/my-flying-box-fr_FR.po` | French translations — edit this |
| `lang/my-flying-box-fr_FR.mo` | Compiled binary used by WordPress — do not edit |

## Scripts

| Script | Purpose |
|--------|---------|
| `bin/translations` | Main entry point — see `./bin/translations help` |
| `bin/manage-translations.py` | Underlying Python tool (`--extract`, `--report`, `--generate-mo`) |
| `bin/fill-translations.py` | Generate an HTML or CSV template for translators |
| `bin/compare-translations.py` | Compare POT and PO to identify new or removed strings |
